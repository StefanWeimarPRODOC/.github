# Issues schreiben — Cheat Sheet

| Prinzip | Erklärung |
|---------|----------|
| **Titel = Ergebnis** | "Realtime-Speichern im Profil-Editor" statt "Speichern verbessern" |
| **Ein Issue = eine Sache** | Lieber zwei kleine Issues als ein überladenes |
| **Akzeptanzkriterien = Checkliste** | Konkret, testbar, abhakbar |
| **Scope-Grenzen setzen** | "Nicht enthalten: ..." verhindert 80% des Scope Creep |
| **Betrifft-Feld ausfüllen** | Gibt Claude Code den Einstiegspunkt ins Codebase |
| **Screenshots > Prosa** | Bei Bugs: ein Bild sagt mehr als drei Absätze |
| **Priorität immer setzen** | Hilft bei der Planung — nicht alles ist Urgent |

## Templates

Beim Erstellen eines Issues auf GitHub erscheint automatisch eine Template-Auswahl:

- **Feature / Enhancement** — Neue Funktionalität oder Verbesserung
- **Bug Report** — Fehlverhalten melden
- **Technical Task** — Refactoring, Infrastruktur, Tooling
- **Documentation** — README, CLAUDE.md, Inline-Docs, Guides
- **Upstream-Diff** — Abweichung vom Upstream-Repository dokumentieren

Die Templates werden zentral über das `.github`-Repository verwaltet und gelten für alle Repos.
